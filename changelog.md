# Changelog

## v0.6.0 (2026-06-12) - 冲突结构性 + 情绪峰值密度 + 投流适配 + 留存曲线 + 区域打分

### 重大更新

**结构维度重构（40分保持）**
- 黄金开局 8→7，第一幕建制 6→5，第二幕中点 8→7，第三幕终局 6→4
- 新增 **冲突结构性（5分）** — 核验冲突是否存在于 premise 中（非人为制造）
- 逃婚杀手跑分：杀手×目标 = S级5分
- 剥离"说谎/隐瞒/误会"核心冲突消失 → 降级

**情绪维度重构（35分保持）**
- CP张力 8→7，情绪波动 5→3，反派 6→5，群像 4→3
- 新增 **情绪峰值密度（5分）** — 行业标准：每90秒一个峰值
- 最长断档不超过3集，需有级联高峰

**市场维度重构（25分保持）**
- 名场面拆分为：**数量与视觉符号（4分）+ 切片分布与剪辑性（3分）**
- 新增：名场面分布均匀度 + 独立成片性检查

**毒舌评审升级**
- 新增 **留存曲线验证（参考，不计分）** — E3≥80%, E7≥60%, E10≥45%
- 新增 **情绪峰值密度复核**
- 新增 **区域调整分（参考）**

**全局新增**
- 目标市场参数（北美/东南亚/拉美/中东），附带区域权重矩阵
- 更新 scoring-anchors-v2.md 同步全部子项调整
- 更新打分自检清单：冲突结构性 + 情绪峰值检查

## v0.5.2 (2026-06-12) - 动机一致性检验 + 完整度置信度标记

### 新增
- **动机一致性检验**（毒舌评审强制项）：逐角色检查核心行动线是否有合理动机，发现断裂和矛盾立即强制扣分
  - 梗概模式上限 -3 分，完整剧本模式上限 -6 分
  - 补充了三个专家的盲区（结构/情绪/市场各自打分，无人检查动机自洽性）
- **完整度置信度标记**：根据输入类型（完整剧本/梗概+大纲/仅logline）标注评分置信度
- **更新 rating-criteria.md**：新增置信度说明表和动机一致性说明
- **更新 critic-prompt.md**：新增 Step 4 动机一致性检验流程，调整最终分公式
- **更新 SKILL.md**：报告格式新增置信度标记和动机一致性报告项
- 最终分公式调整为：min(专家总分 - 毒舌常规扣分 - 动机一致性扣分, 毒舌总分)

## v0.5.1 (2026-06-12) - 制作可行性降级为参考信息

### 修复
- **降级** 制作可行性从评分维度 → 参考信息，**不参与评分**
- **调整理由：** 剧本评审skill应以剧本本身（故事/结构/情绪/市场）为核心判断，执行能力不应影响评级
- **更新 rating-criteria.md：** 市场维度降回4个子维度，制作可行性移至"参考信息"区
- **更新 critic-prompt.md：** 移除制作可行性复核的扣分逻辑，改为信息参考
- **更新 SKILL.md：** 报告格式中标记为"不计入评分"

## v0.5.0 (2026-06-12) - 梗概模式 + 单列评分 + 制作可行性

### P0: 新增梗概模式（Quick Scan）
- **新增** 输入级别设计 — 模式A（完整剧本 Deep Dive）vs 模式B（梗概 Quick Scan）
- **新增** 梗概模式触发逻辑：用户仅提供 logline+人物小传+大纲时自动降级，不跑文件预处理和 Layer 1 提取
- **新增** 梗概模式输出标注"基于梗概"，完整剧本产出后建议重评
- **限制** 校准模式仅支持完整剧本输入

### P1: 修正最终分计算公式
- **修正** 公式从 `max(专家总分 - 毒舌扣分, 毒舌总分)` → `min(专家总分 - 毒舌扣分, 毒舌总分)`
- 原则：从严评审，毒舌已经基于专家分做了逐项复核，取 min 避免分数膨胀
- 同步更新 critic-prompt.md v3 输出格式

### P2: 移除执行潜力分
- **移除** rating-criteria.md 中的"双列版"设计和执行潜力分列
- **移除** SKILL.md 中的双列评分表输出格式
- **移除** scoring-anchors-v2.md 中的"我是给已实现还是潜力打分"自检项
- **备注** 执行潜力日后可通过校准模式结合运营/社媒实际数据反馈来驱动

### P3: 新增制作可行性维度
- **新增** rating-criteria.md 中的"制作可行性评估"（5分），替换原来的"视觉执行评估"
- **新增** 成本预估公式：基准 1 元/秒（成片），精品剧推荐片比 1:4，量产剧推荐片比 1:2
- **新增** critic-prompt.md 中的"制作可行性复核"步骤（Step 4）
- **新增** 毒舌评审报告中的"制作可行性复核"区块

### P4: 移除硬编码模型指定
- **移除** SKILL.md 中所有 `model: "qwen-turbo"` / `model: "qwen-plus"` 的强制指定
- **改为** 按执行时的 OpenClaw 默认配置走，附使用建议（完整剧本可选轻量模型降低成本）

### v0.3.0 (2026-06-09) - 反虚高版

### 新增
- **毒舌评审 Agent** `core/critic-prompt.md` — 专门挑刺、强制扣分的第四位专家
- **评分锚点 v2** `core/scoring-anchors-v2.md` — 提高标准+行业基准线，80分=前15%
- **高分警报** `scripts/validate_scores.sh` — 总分≥85 自动触发复核

### 改进
- **打分标准提高** — 行业平均分 65-70，80分=前15%，85分=前5%
- **CP张力区分单向/双向** — 单向偏爱最多6分，7分以上必须双向奔赴
- **视觉执行评估去掉成本限制** — 只看AI执行效果
- **打分自检清单** — 每个专家打分前必查5个问题
